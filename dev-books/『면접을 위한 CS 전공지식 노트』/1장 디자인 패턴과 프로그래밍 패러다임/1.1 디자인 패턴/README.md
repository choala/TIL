## 디자인 패턴(Design Pattern)
> 프로그램을 설계할 때 발생했던 문제점들을 객체 간의 상호 관계 등을 이용하여 해결할 수 있도록 하나의 '규약' 형태로 만들어 놓은 것
- 쉽게 말하면, 소프트웨어 디자인 과정에서 자주 발생하는 문제들에 대한 전형적인 해결책
- 객체 지향 설계에 주로 사용되며 설계 과정과 개발자간 의사소통에 이점을 제공함
- 디자인 패턴에 대한 더 자세한 설명은 https://refactoring.guru/ko/design-patterns

<br>
